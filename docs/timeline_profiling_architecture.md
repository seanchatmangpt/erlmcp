## Use Cases

### 1. Tool Performance Analysis

```erlang
%% Profile complete tool lifecycle
{ok, Result, Timeline} = erlmcp_profiler:profile_tool_call(
    <<"fs">>,
    <<"write_file">>,
    #{path => <<"/tmp/large.txt">>,
      content => crypto:strong_rand_bytes(1024 * 1024)},
    #{max_events => 50000}
).

%% Analyze bottlenecks
#{event_counts := #{
    scheduler := SchedCount,
    port := PortCount,
    gc := GCCount
}} = maps:get(statistics, Timeline).

%% Expected: PortCount > 0 (file I/O)
%% Warning: GCCount > 5 (memory fragmentation)
```

### 2. Regression Detection

```erlang
%% Establish baseline
{ok, _, Baseline} = erlmcp_profiler:profile_timeline(
    fun critical_operation/0,
    <<"baseline_v1.0">
),
ok = file:write_file("baseline.json",
    jsx:encode(Baseline)).

%% Test new version
{ok, _, Current} = erlmcp_profiler:profile_timeline(
    fun critical_operation/0,
    <<"current_v1.1">
).

%% Automated comparison
Diff = erlmcp_profiler:compare_profiles(Baseline, Current),
#{duration_percent_change := Change} = maps:get(diff, Diff),

case Change > 10.0 of
    true -> error({performance_regression, Change});
    false -> ok
end.
```

### 3. Transport Comparison

```bash
# Profile stdio
erlmcp profile:transport stdio 1024

# Profile tcp
erlmcp profile:transport tcp 1024

# Compare
erlmcp profile:compare profile_stdio_*.json profile_tcp_*.json

# Expected result: TCP has ~2x overhead but better scalability
```

### Optimization Strategies

1. **Event Sampling**: `sample_rate => 0.1` (10% of events)
2. **Event Filtering**: `include_gc => false` (skip GC)
3. **Max Events**: `max_events => 1000` (limit collection)
4. **Short Duration**: Profile specific operations only
