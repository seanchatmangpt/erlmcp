VULNERABILITY #1: DNS Rebinding Attack
  Severity: CRITICAL
  Location: All HTTP transport endpoints
  Impact: Remote attackers can access local MCP server via JavaScript
  Data at Risk: Resources, tools, prompts, execution results
  Example Attack:
    1. Attacker hosts malicious.com
    2. Sets DNS to resolve to 127.0.0.1
    3. JavaScript in victim's browser makes requests to 127.0.0.1:8080
    4. No Origin header validation = attacker gains access
  Fix Time: 4-6 hours
  Mitigation: Implement Origin header validation (Gap #3)

VULNERABILITY #2: Session Hijacking
  Severity: CRITICAL
  Location: HTTP, SSE, WebSocket transports
  Impact: Clients cannot identify or resume sessions
  Data at Risk: All client-server communication
  Problem: No MCP-Session-Id header, no session state tracking
  Fix Time: 10-12 hours
  Mitigation: Implement session management with secure ID generation (Gap #2)

VULNERABILITY #3: MITM Attacks
  Severity: HIGH (elevated to CRITICAL with local deployments)
  Location: sys.config, HTTP server binding
  Impact: All communication in plaintext
  Data at Risk: All client-server communication
  Problem: No HTTPS enforcement, HTTP servers may bind to 0.0.0.0
  Fix Time: 4-6 hours
  Mitigation: Enforce HTTPS, bind to 127.0.0.1 only (Gap #21)
